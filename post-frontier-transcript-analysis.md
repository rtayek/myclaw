## Key Insights from `transcript.txt`

### 1. The Post-Frontier Reality: The Harness is the Product
* **Monolithic Models are Not the Product**: Raw model capabilities are converging. The true commercial and user value lies in the **harness, orchestration loop, context layer, and evaluation harness**.
* **Task-Specific Evaluation Loops**: Enterprises (e.g., DoorDash) and individual developers are building custom eval harnesses on their own real-world codebases rather than relying on public LLM benchmarks.
* **Specialized Model Pairing**: No single model excels at every capability or cost point. High-value systems orchestrate fast, cheap local models alongside specialized tool callers and remote frontier advisors.

### 2. Open-Weight Dominance & Compute Economics
* **The 90% Open-Token Shift**: Benchmark's Peter Fenton predicts that **90%+ of tokens will be generated by open-weight models** within 18 to 24 months.
* **Price of Compute vs. Frontier Markups**: Open-weight models run at raw compute costs without the 3x to 5x API markups charged by cloud labs.
* **Performance-Driven Fine-Tuning**: Applied AI companies adoption of open models is driven primarily by **performance, lower latency, and domain fine-tuning**, with lower cost acting as an added benefit.

### 3. Hybrid & Deskside Local Compute
* **The Power Bottleneck**: Data center growth is constrained by power grid capacity.
* **Deskside AI Hardware**: Local devices with unified CPU/GPU memory (e.g., NVIDIA DGX Spark architectures) allow running efficient 35B–70B open models locally at low wattage (a few hundred watts).
* **Local Orchestrator + Remote Advisor**: Local open models handle everyday queries, prompt processing, and tool calls at zero per-token cost with zero data leakage. When deep reasoning is required, the local harness escalates to remote frontier models (Claude, Gemini, GPT) as sub-agents.

### 4. Data & Context Sovereignty
* **Tacit Knowledge as Economic Defensibility**: User and enterprise value lives in proprietary context, private workflows, and internal data. Surrendering context data to closed APIs destroys long-term defensibility.
* **Portability via Context Rehydration**: Switching models is simply a matter of rehydrating local context records. Owning conversation history on disk guarantees absolute provider independence.

### 5. Efficiency Metric: Token Value per Watt
* Shift from parameter count to **Token Value per Watt per User**—delivering maximum actionable output ROI per unit of power and compute consumed.
