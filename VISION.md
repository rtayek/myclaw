# Product Vision & Accessibility

## Core Mission
MyClaw is designed from the ground up to empower blind, low-vision, deaf, and hard-of-hearing developers. Accessibility is the foundational architecture of the platform, not an add-on feature.

## Core Pillars

### 🗣️ Multi-Sensory Collaboration & Voice Control
* **Speech Input & Output:** Integrated dictation for hands-free prompting and speech synthesis (TTS) for spoken answers.
* **Audio Sonification:** Instant translation of background events, test results, and stream states into distinct tonal cues.

### ⌨️ Zero-Mouse Keyboard Navigation
* **Keyboard-First Flow:** Complete elimination of mouse precision requirements, drag-and-drop mechanics, and tiny web UI controls.
* **Global Context Capture:** Hotkey text buffer ingestion (`Win+V`) directly from the native editor into the AI runtime.

### 👁️ Visual Serenity & High Contrast
* **Readable Consoles:** Scalable typography, high-contrast themes, and high-legibility Markdown text streams.
* **Screen Reader Optimization:** Outputs clean text streams parsed directly by screen readers from the first character.

### 🖥️ Spatial Layout (Multi-Monitor Allocation)
* **Left Screen (Engine):** High-visibility shell panel managing automated background execution streams.
* **Center Screen (Construction Deck):** Native Eclipse Java IDE workspace.
* **Right Screen (Visual Library):** Markdown documentation and transcript viewer without chaotic layout clutter.
